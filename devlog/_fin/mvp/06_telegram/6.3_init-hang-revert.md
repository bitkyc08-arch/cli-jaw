# 6.3 bot.init() hang → bot.start() 단독 사용으로 복귀

## 경위

1. 최초 구현: `bot.start()` 내부에서 자동 init → **작동** (`@Clawcli_bot polling active`)
2. 디버그 중 `bot.init()` 명시 분리 → `await bot.init()` **무한 hang**
3. `apiThrottler()`를 init 전에 적용한 것도 hang 원인 의심
4. throttler 순서 바꿔봐도 `bot.init()` 자체가 1분+ hang

## 원인

`bot.init()`는 `getMe` API 호출인데, 단독 호출 시 grammy 내부에서 connection 설정 차이가 있는 듯.
`bot.start()` 내부에서는 polling과 init을 동시에 처리하면서 정상 작동.

## 수정

```diff
-async function initTelegram() {
-    await bot.init();          // ← 무한 hang
-    bot.api.config.use(apiThrottler());
+function initTelegram() {
+    // bot.start()가 init 자체 처리
     bot.start({
         drop_pending_updates: true,
-    });
+        onStart: (info) => console.log(`[tg] ✅ @${info.username} polling active`),
+    });
```

- `apiThrottler()` 제거 (hang 의심)
- `bot.init()` 제거 → `bot.start()` 내부 init 사용
- `onStart` 콜백으로 성공 확인
