# Phase 5.6: Sub-Agent 프롬프트 격리

## 문제

`buildArgs()`가 `getSystemPrompt()`를 하드코딩 → sub-agent에 오케스트레이션 규칙이 전달됨 → sub-agent가 실행 대신 JSON subtask를 뱉음.

## 수정

### 1. `buildArgs(cli, model, effort, prompt, sysPrompt)`
시스템 프롬프트를 매개변수로 받도록 변경. claude는 `--append-system-prompt`, gemini는 `--system-instruction`에 전달.

### 2. `spawnAgent`에서 sysPrompt 분기
- main agent: `getSystemPrompt()` (orchestration 포함)
- sub-agent (forceNew): `customSysPrompt` (역할 전용)

### 3. Sub-agent 시스템 프롬프트
```
당신은 "{이름}" 입니다.
역할: {역할}

## 규칙
- 주어진 작업을 직접 실행하고 결과를 보고
- JSON subtask 출력 금지 (실행자 ≠ 기획자)
- 자연어로 간결하게 보고
- 사용자 언어로 응답
```
